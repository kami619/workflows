# Quality Analysis

Run a comprehensive quality assessment on the repository in this session.

## Process

1. Read the repository structure to identify type, language, and framework
2. Execute the 7-dimension analysis as defined in CLAUDE.md
3. Score each dimension 0-10 with evidence
4. Calculate weighted overall score
5. Identify critical gaps, quick wins, and prioritized recommendations
6. Write the full report to `artifacts/quality-repo-analysis/quality-report.md`

## Instructions

- Start by listing the repository root and key directories (.github/workflows, test/, e2e/, Dockerfile, Makefile)
- For each dimension, cite specific files and line numbers as evidence
- Compare findings against the gold standards defined in CLAUDE.md
- Be thorough but pragmatic -- focus on what matters most for this type of repository
- If a dimension is not applicable, mark it N/A and explain why

Begin the analysis now.
